# OpenSpec 提案：网站现代化重构 (Proposal: Website Redo)

## 1. 意图 (Intent)
当前的个人网站“光 · 学习空间”虽然在视觉上实现了赛博玻璃拟态的炫酷效果且具备文章历史追溯功能，但在工程结构上属于单页面/单文件（所有逻辑糅杂在 `app.js`，无构建工具支持）。
为了保证长期可维护性、提升加载性能，并严格遵循 **规范驱动开发 (SDD)**，本项目将使用 OpenSpec 工作流进行完全重做。

## 2. 技术栈架构 (Technical Architecture)
为了兼顾“KISS 原则（简洁至上）”和“现代化工程”，特在此提案中提出以下选型方案（请您定夺）：

- **方案 A（推荐）：Vite + 原生工程化 (Vanilla JS/TS + Modules)**
  - **核心**：保留无框架的轻量级纯粹感，但引入 Vite 作为打包工具。
  - **模块化**：将臃肿的 `app.js` 拆分为 `router.js`、`api.js`、`ui.js` 和 `particles.js`等。
  - **样式**：引入 CSS Variables 拆分模块（如 `hero.css`, `cards.css`），方便管理炫彩的玻璃拟态特效。
  
- **方案 B：现代化完整框架 (如 React / Vue / Next.js)**
  - 适合未来可能增加复杂交互（如用户系统、评论系统）的扩展预期。

*(在获得您确认前，默认依照方案 A 准备。)*

## 3. 核心规范基线 (Source of Truth Baseline)
所有旧版核心需求必须在重设中达到 100% 还原：
1. **暗色极客美学**：全局深色背景，霓虹发光节点与玻璃拟态（Glassmorphism）高斯模糊质感卡片。
2. **交互性粒子特效**：首页 Canvas 驱动的星空鼠标引力互动，且要求抽离为独立组件。
3. **响应式 SPA 路由**：首页、学习笔记、知识搜索三大视图的丝滑切换（不刷新）。
4. **历史记录时间机**：读取 `articles.json` 的 `history` 数组完成老版本文章的追溯展示弹窗。
5. **SEO 最佳实践**：合理使用 `<main>`、`<article>` 语义化标签，并利用构建工具压缩首屏。

## 4. 行动计划 (Execution Plan)
> *注：此路线图遵循 OpenSpec 工作流的 Review & Apply 循环机制。*

1. **[提案阶段]** 等待您对本重构提案（以及技术栈选择）的审批。
2. **[定义阶段]** 创建 `openspec/specs/main.md` 作为项目唯一的绝对事实来源（Source of Truth）。
3. **[实施阶段]** 清除重金属代码，全新初始化 Vite 骨架，并逐模块从底向上迁移样式与逻辑。
4. **[验证阶段]** 构建产物后开启本地预览进行 UX 核查。
5. **[归档阶段]** 重做通过后合并记录并自动 Push 发版。
