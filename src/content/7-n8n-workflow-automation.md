## 一、n8n 是什么？

n8n（读作 "nodemation"）是一个**开源、可自托管的低代码工作流自动化平台**。它允许用户通过可视化拖拽界面连接不同的应用、API 和服务，实现重复性任务的自动化。

**核心定位**：介于纯无代码工具（如 Zapier）和纯编程之间，提供「可视化 + 可编码」的混合模式。

## 二、核心概念

### 2.1 Nodes（节点）

Node 是 n8n 工作流的基本构建单元，每个 Node 执行一个具体操作：

- **应用节点**：与外部服务交互（Gmail、Slack、Google Sheets、GitHub 等），500+ 预置集成
- **核心节点**：处理逻辑控制（IF 条件判断、Switch 分支、Merge 合并、Loop 循环）
- **函数节点**：允许编写 JavaScript/Python 自定义逻辑
- **AI Agent 节点**：集成 LLM、向量数据库、Embedding 等 AI 能力

每个节点有**输入**和**输出**，数据从一个节点流向下一个节点，形成数据管道。

### 2.2 Triggers（触发器）

触发器是工作流的起点，决定了「什么时候开始执行」：

- **Webhook 触发**：监听外部 HTTP 请求，收到请求立即执行
- **定时触发（Cron）**：按计划执行（每小时/每天/每周）
- **应用事件触发**：当第三方应用发生事件时触发（如收到新邮件、新表单提交）
- **手动触发**：点击按钮手动运行，适合测试和调试
- **表单触发**：生成一个表单页面，用户提交后触发工作流

### 2.3 Webhooks（网络钩子）

Webhook 是一种让外部服务主动推送数据给 n8n 的机制：

1. n8n 生成唯一的 Webhook URL
2. 外部服务配置该 URL 作为回调地址
3. 当事件发生时，外部服务 POST 数据到该 URL
4. n8n 接收数据并触发后续工作流

**安全机制**：支持 Basic Auth、Header Auth、JWT 认证。

## 三、与同类工具对比

| 维度 | n8n | Zapier | Make |
|------|-----|--------|------|
| **开源** | ✅ 完全开源 | ❌ | ❌ |
| **自托管** | ✅ Docker 部署 | ❌ 仅云端 | ❌ 仅云端 |
| **价格** | 社区版免费 | 按任务计费，较贵 | 按操作计费 |
| **代码灵活性** | JS/Python 自由编写 | 受限 | 受限 |
| **AI 集成** | 原生 AI Agent 节点 | 有限 | 有限 |
| **学习曲线** | 中等（需基础技术能力） | 低 | 中 |
| **数据隐私** | 数据完全可控 | 数据在第三方 | 数据在第三方 |

**结论**：如果你重视数据隐私、需要复杂逻辑、或想免费使用 → 选 n8n。

## 四、AI Agent 工作流（2025 重点方向）

n8n 在 2025 年全面拥抱 AI-first 策略，提供 70+ AI 节点：

### 4.1 支持的 AI 能力
- **LLM 调用**：OpenAI (GPT)、Google Gemini、Claude、Hugging Face
- **RAG（检索增强生成）**：连接向量数据库（Pinecone、Qdrant），实现文档问答
- **多 Agent 协作**：Retriever Agent 检索 → LLM Agent 生成 → Verifier Agent 验证
- **工具调用**：AI Agent 可自主调用 API、查询数据库、发送邮件

### 4.2 典型 AI 工作流示例

**示例 1：智能客服**
收到工单 → AI 分类意图 → 查知识库 → 生成回复 → 发送

**示例 2：研究助手**
输入问题 → 搜索多个源 → AI 摘要 → 事实核查 → 输出报告

**示例 3：自动化内容创作**
监控 RSS → 提取热点 → AI 写稿 → 审核 → 发布到 WordPress

## 五、自托管部署（Docker）

### 前置要求
- Docker Engine 已安装
- 至少 2GB RAM（建议 4GB）
- 10GB 磁盘空间

### 最简启动命令
```bash
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n
```

启动后访问 `http://localhost:5678` 即可使用。

### 生产环境建议
- 使用 Docker Compose + PostgreSQL 做持久化存储
- 配置 HTTPS 反向代理（Nginx/Caddy）
- 设置 Basic Auth 或 SSO 认证

## 六、实际应用场景汇总

| 场景 | 工作流描述 |
|------|----------|
| **GitHub 自动化** | PR 创建 → 自动 Code Review → Slack 通知 |
| **邮件处理** | 收到邮件 → AI 分类 → 自动回复/转发 |
| **数据同步** | 数据库变更 → 同步到 Google Sheets → 生成报告 |
| **社交媒体** | 定时抓取热点 → AI 生成内容 → 多平台发布 |
| **财务报表** | 定时查询系统 → AI 汇总 → 生成 PDF → 邮件发送 |
| **日程管理** | 自然语言输入 → AI 解析 → 创建日历事件 |

## 七、学习路线建议

1. **入门**：官网交互式教程 → 熟悉 UI 和基本节点
2. **实战**：搭建一个「邮件通知 + Google Sheets 记录」的简单工作流
3. **进阶**：学习 Webhook、HTTP Request 节点，连接自定义 API
4. **AI 集成**：试用 AI Agent 节点，搭建 RAG 问答系统
5. **生产部署**：Docker 自托管 + PostgreSQL + Nginx 反向代理

## 八、关键要点总结

1. n8n 是开源的 Zapier 替代品，最大优势是**可自托管**和**代码灵活性**
2. 核心三要素：Nodes（做什么）、Triggers（什么时候做）、Webhooks（怎么接收外部数据）
3. 2025 年重点方向是 **AI-first 自动化**，70+ AI 节点覆盖 LLM、RAG、多 Agent
4. Docker 一行命令即可启动，上手门槛低
5. 适合开发者和技术团队构建复杂的、数据敏感的自动化流程
