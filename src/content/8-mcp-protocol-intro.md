## 一、MCP 是什么？

Model Context Protocol（MCP，模型上下文协议）是 Anthropic 于 2024 年 11 月推出的**开放标准**，旨在标准化 AI 系统（尤其是 LLM）与外部工具、数据源的交互方式。

**一句话理解**：MCP 就是 AI 的 USB-C 接口 —— 一个统一的标准，让任何 AI 模型都能连接任何外部服务，无需为每个组合写定制代码。

### 解决了什么问题？

在 MCP 之前，每个 AI 应用要连接 N 个数据源和 M 个工具，就需要 N×M 个定制连接器。MCP 将这个 O(N×M) 问题简化为 O(N+M) —— 每个工具只需实现一次 MCP Server，每个 AI 应用只需实现一次 MCP Client。

## 二、核心架构

MCP 采用 **Client-Host-Server** 三层架构，灵感来自 LSP（Language Server Protocol）：

### 2.1 三个角色

| 角色 | 说明 | 示例 |
|------|------|------|
| **Host（宿主）** | 用户直接交互的 AI 应用 | Claude Desktop、IDE、聊天界面 |
| **Client（客户端）** | 宿主内部的协议桥梁，发现并连接 Server | 内嵌在 Host 中 |
| **Server（服务端）** | 暴露具体能力的外部服务 | 文件系统、数据库、GitHub API |

### 2.2 通信协议

所有通信基于 **JSON-RPC 2.0**，支持两种传输方式：
- **stdio**：本地进程通信（适合桌面应用）
- **HTTP + SSE**：远程服务通信（适合云端部署）

一个 Host 可以同时连接多个 Server，实现能力组合。

## 三、核心三要素

### 3.1 Tools（工具）

Server 暴露的**可执行函数**，由 AI 模型主动调用：
- 数据库查询、API 调用、文件操作、代码执行
- 每个 Tool 有名称、描述、输入/输出 Schema（JSON 格式）
- **模型控制**：LLM 根据上下文自主决定调用哪个 Tool
- 敏感操作支持人工审批（Human-in-the-loop）

### 3.2 Resources（资源）

Server 提供的**上下文数据**，由应用层决定如何使用：
- 文件内容、数据库记录、API 响应、日志、截图
- 每个 Resource 有唯一 URI 标识
- 支持文本和二进制数据
- **应用控制**：由 Host 应用决定注入哪些上下文

### 3.3 Prompts（提示模板）

可复用的指令模板，引导 AI 交互：
- 预定义的工作流模板
- 可参数化的提示词
- 标准化的交互模式

## 四、工作流程

MCP 的典型交互流程：

1. **工具发现**：LLM 查询 MCP Server，获取可用工具列表
2. **Schema 理解**：Server 返回每个工具的输入/输出 Schema
3. **结构化调用**：LLM 构造 JSON 请求，调用目标工具
4. **外部处理**：Server 执行操作（查询数据库、调用 API 等）
5. **结果返回**：Server 以标准格式返回结果
6. **上下文整合**：LLM 将结果整合进回答

## 五、实际应用场景

### 5.1 编程助手（最广泛的场景）
- Claude Desktop / Cursor / Zed 等 IDE 通过 MCP 访问代码仓库
- 实时获取项目上下文，而非仅依赖静态训练数据
- 自动化 PR Review：拉取变更 → AI 分析 → 生成审查意见

### 5.2 自动化工作流
| 场景 | MCP 工作流 |
|------|----------|
| 智能客服 | 收到工单 → 查知识库 → 生成回复 |
| 研究助手 | 输入问题 → 搜索多源 → AI 摘要 → 输出报告 |
| 代码重构 | 扫描仓库 → 识别热点 → 生成小 diff → 逐步重构 |
| 数据分析 | 连接数据库 → 查询数据 → AI 生成可视化报告 |

### 5.3 企业集成
- 连接内部系统（CRM、ERP、知识库）
- 数据不离开企业网络（自托管 Server）
- 统一的权限管理和访问控制

## 七、与类似概念的对比

| 开源 | ✅ 开放标准 | - | ✅ |

## 八、关键要点总结

1. MCP 是 AI 世界的 **USB-C 接口**，解决了 AI 与外部工具的 N×M 集成难题
2. 三层架构：Host（用户界面）→ Client（协议桥梁）→ Server（能力提供方）
3. 核心三要素：**Tools**（AI 调用的函数）、**Resources**（上下文数据）、**Prompts**（交互模板）
4. 基于 **JSON-RPC 2.0** 通信，灵感来自 LSP
5. 2025 年被主流厂商全面采纳，已成为 AI Agent 生态的基础设施标准
6. 最大价值：**写一次 Server，所有 AI 都能用；写一次 Client，连接所有工具**
