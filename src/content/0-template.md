## 🔬 一、 核心原理解析 (理解层)

在建立直觉之后，这里开始上专业硬核内容。

### 1. 专业定义
说明它在学术界或工程界的严谨定义。
*例如：JSON Web Token 是一个开放标准 (RFC 7519)，它定义了一种紧凑的、自包含的方式，用于作为 JSON 对象在各方之间安全地传输信息。*

### 2. 第一性原理推导 / 结构拆解
拆解它的构成。
- **Header（头部）**：描述签名算法。
- **Payload（负载）**：存放数据（如 UserID）。
- **Signature（签名）**：防止被篡改的核心。

（可以配合适当的加粗、表格来增强结构感）
